Object-safe Traits

Mon 17 August 2015
By Kieran Hunt

Traits are one of Rust's hidden super powers. Creating objects using traits can be done if your traits are object-safe. The definition of an object safe trait is a trait which has no methods which take or return a value dependent on the implementor's own type. Its type must be known at compile time. This restriction is in place so that the compiler may make space for the object on the stack; this is not possible without knowing what the type of the object is.
